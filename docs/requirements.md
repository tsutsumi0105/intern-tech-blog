# テックブログ制作のご依頼

## ご依頼の背景

私たちの会社では、エンジニアが日々の学びや技術的な気づきを発信するための「テックブログ」を新しく立ち上げたいと考えています。

社内には技術に詳しいメンバーがいるのですが、なかなかアウトプットする場がなく、せっかくの知見が社内に留まってしまっている状態です。社外に向けて発信することで、技術コミュニティへの貢献や、採用ブランディングにもつなげたいと思っています。

今回、このブログサイトの制作をお願いしたいです。

---

## どんなブログにしたいか

### コンセプト

「エンジニアの学習記録を、社外にも発信できるテックブログ」

インターン中に学んだことや、技術Tips、つまずいたポイントの解決方法など、実際の学習・開発の中で得た知見を記事にして発信していきます。

### 読者のイメージ

- 技術に興味のあるエンジニア（社外の方がメイン）
- プログラミングを学び始めた初学者
- 同じ技術で困っている人

### 書き手について

- 最初はインターン生が記事を書きます
- 将来的には社員も記事を書けるようにしたいです
- 記事の投稿や管理は、専用の管理画面（CMS）から行います。ブログのサイト自体にログイン機能などは不要です

---

## ページ構成のイメージ

以下のようなページがほしいです。

### 1. トップページ

ブログの顔となるページです。最新の記事がいくつか並んでいて、ブログ全体の雰囲気が伝わるようなイメージです。

- 最新の記事を数件、「タイトル」「公開日」「カテゴリ」「概要文」「サムネイル画像」のカード形式で表示
- 記事をクリックすると、その記事の詳細ページに遷移する

### 2. 記事一覧ページ

すべての記事をまとめて見られるページです。トップページには最新の数件しか載せないので、**過去の記事も含めて全部見たい**ときはこのページを見るイメージです。

- 全記事を**新しい順**に表示
- トップページと同じカード形式で並べてほしい

### 3. 記事の詳細ページ

記事を1本まるごと読めるページです。

表示してほしい情報：

- 記事のタイトル
- 公開日
- カテゴリ（クリックするとそのカテゴリのページに飛べると嬉しい）
- サムネイル画像
- 本文

記事の本文は、管理画面のリッチエディタで書いたとおりに、見出し・太字・リスト・コードブロック・画像などの装飾がきちんと反映された状態で表示してほしいです。

### 4. カテゴリ別の記事一覧ページ

特定のカテゴリに属する記事だけを表示するページです。

- ページの上部に、そのカテゴリの**画像・名前・説明文**をヘッダーとして表示してほしい
  - 例：Reactのページなら、Reactっぽいイメージ画像と「Reactに関する学習記録やTipsをまとめています」のような紹介文
- その下に、そのカテゴリの記事がカード形式で並ぶ
- URLは `/category/react` や `/category/typescript` のような、**カテゴリの英語名がURLに入る**きれいな形にしたい（数字やIDではなく）

想定しているカテゴリの例：

- JavaScript
- React
- TypeScript
- Next.js
- Firebase
- Git / GitHub
- その他

カテゴリは後から増やしたり、名前や説明文・画像を変えたりすることがあります。そのたびにエンジニアにコードを修正してもらうのは避けたいので、**管理画面から自由にカテゴリの追加・編集ができる**ようにしてほしいです。

## デザインについて

デザインは**Figma**というツールで別途お渡しします。基本的には、そのデザインに沿って作っていただければと思います。

全体のイメージとしては：

- **シンプルで読みやすいデザイン**
- 余白をしっかり取って、文字がぎゅうぎゅうにならないように
- テックブログらしい、すっきりした印象

---

## スマートフォン対応

PCだけでなく、**スマートフォンでも見やすい**ようにしてください。最近はスマホで記事を読む人も多いので、これは必須でお願いします。

---

## 記事の管理について

記事やカテゴリの投稿・編集・削除は、**microCMS**というサービスの管理画面から行う予定です。ブログサイト自体には記事を書く機能やカテゴリを追加する機能は不要です。

## あると嬉しい機能

以下は必須ではありませんが、あると嬉しいなと思っている機能です。

- **ページネーション**：記事が増えてきたときに、1ページに表示する件数を区切って「次のページ」「前のページ」で移動できる機能
- **目次**：記事の詳細ページに、見出しから自動で作られる目次があると、長い記事でも読みやすい
- **OGP対応**：記事のURLをSNS（XやSlackなど）に貼ったときに、タイトルやサムネイルがカード形式で表示される機能
- **パンくずリスト**：「トップ > カテゴリ > 記事名」のように、今どのページにいるかがわかるナビゲーション

全部やってほしいというわけではなく、2週間という期限の中でどれなら対応できそうか、**最初にご提案いただけると嬉しいです**。優先度や工数感を踏まえて、一緒に決めましょう。

---

## 不要なもの

以下の機能は今回不要です：

- コメント機能（読者がコメントを書く機能）
- RSS配信
- ログイン機能（記事の管理はmicroCMSで行うため）
- 会員登録機能
